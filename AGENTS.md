# AGENTS.md — 皮克斯短剧生产线 · 通用 agent 约定

> 跨工具通用约定文件。任何能读 `AGENTS.md` 的 agent / IDE（Antigravity、其他 AI 编辑器等）都应把本文件当作本仓库的最高约束。
> Claude Code 用户：运行时机制（子代理委派 / slash 命令 / 工具权限隔离）定义在 `CLAUDE.md` 与 `.claude/`，那是源真相；本文件是其 **tool-agnostic 镜像**，供不识别 `.claude/` 的工具使用。

## 这是什么

一条"从一句话创意到成片蓝图"的皮克斯式短剧生产线。核心架构（模型/工具无关）：
**编排器**按阶段委派**专职角色（persona/agent）**，每阶段产出一份落盘文档；其中"脑信任评审"是**评审↔改写循环**——评审只诊断、不改稿，改写交给另一角色。

各角色的完整系统提示在 `.claude/agents/*.md`，方法论知识库在 `.claude/skills/pixar-method/SKILL.md`。**任何工具都可把这些 markdown 当作角色定义 / 上下文直接读取。**

## 五条硬指令（不可违反）

1. **先情感命题，再情节。** 每个剧先写一句"最疼的人类处境"（不是剧情梗概）。任何不能加深这句话的桥段一律删。
2. **尽早可视化、尽早暴露问题。** 早期允许丑/乱/蠢（烂版本文化）。禁止用"还不成熟"拖延出版本。
3. **评审只诊断，不夺权。** 评审只说"哪里没看懂/不信/不在乎/拖"，禁止代写桥段。改法由改写角色或导演定。
4. **角色用 Want/Need + 缺陷自驱。** 危机必须由主角偏执放大，禁天降误会。
5. **催泪靠前文锚点回收。** 结尾不准发明新信息煽情，强情绪点必来自前文已有的物件/关系/记忆。

## 反模式红线（出现即不通过）

被动主角 / 二元脸谱反派 / 偶然偷听等廉价信息获取 / 巧合或机械降神帮主角脱困（巧合只能让主角陷麻烦）/ 用台词硬讲主题、结尾像总结作文 / 一个世界塞多条超现实设定 / 角色塞太多标签。

## 九阶段流水线（串行，每关有"通过红线"，不过不进下一关；每关停下等导演确认）

| 阶段 | 角色（系统提示文件） | 产物 | 通过红线 |
|---|---|---|---|
| 1 选题 | what-if-generator | premise.md | 可视化 + 会做错事的主角 + 天然困境 |
| 2 角色 | character-architect | characters.md | Want可拍/Need可感/对抗 + 有创伤 |
| 3 世界观 | world-builder | world.md | 1条核心假设 + 规则推动剧情 + 代价红线 |
| 4 结构 | story-spine | beats.md | 因果链 / 无巧合解围 / 非被动主角 |
| 5 情绪 | emotion-engineer | emotion.md | 泪点回收锚点 + 双层编码 |
| 6 评审 | braintrust-critic ↔ reviser | braintrust_log.md | 四视角全部"成立" |
| 7 分镜 | storyboard-director | storyboard.md | 静音可懂 + 一镜一任务 |
| 8 生产 | production-spec | production/（style/assets/shots/audio/edit/checklist） | 锁参考图后逐镜生成 |
| 9 质检 | continuity-checker | review.md | 一致性 OK + 结尾完成主题回收 |

> 进入第 7 阶段（分镜）前，必须对 beats + emotion 跑一次完整脑信任评审。

## 文件即记忆

角色之间上下文隔离。**所有阶段产物必须落盘到 `productions/<剧名>/`**。每个角色开工先读该阶段所需的上游文件（不通读全目录），结束把成果写回固定文件名。新建一部剧：复制 `productions/_template/` 为 `productions/<剧名>/`。

## 效率与 token 纪律

1. 只读所需文件，不通读全目录。
2. 大体量产物（分镜）按单元产出、编排器拼接落盘，**不要反复读写正在增长的整份大文件**（O(n²) 成本）。
3. 单次产出控制体量（如单集），避免超时。
4. 评审 / 质检角色为只读：只回报诊断，由编排器/改写角色落盘。
5. 跨文件命名一致（同一物件/角色全程同一英文名）。
6. 风险处置写成工具可执行指令（如 `[GENERATE: …]` / `[POST-EDIT REDLINE: …]`），不动剧情。

## 置信度纪律

脑信任、Want/Need、烂版本文化、推倒重来为**高置信**一手材料，作硬约束。「痛苦测试 / 泪点锚定 / 单一假设」为对皮克斯公开做法的**归纳**提法，作可调技法，非官方原术语。不要把归纳包装成皮克斯铁律。
